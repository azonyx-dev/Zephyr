# Contributing

## Getting started

1. Fork the repository and create a branch from `main`.
2. Make your changes against a clean build.
3. Open a pull request with a clear description of what changed and why.

## Code style

- C++20 throughout, no extensions.
- Keep headers self-contained.
- No unnecessary dependencies.

## Pull requests

- One logical change per PR.
- Describe what the change does, not how the code works.
- If you are fixing a bug, include a minimal reproduction in the description.
